# 파일 ▸ 내보내기 — 분석 보고서

## 1. 개요

`js/export.js`는 "파일 ▸ 내보내기" 메뉴 전체(JSON 선택 내보내기, 전체 백업, DDL 생성/정규화, PNG 이미지(섹션 포함/제외), SVG, Markdown, HTML, 인쇄/PDF, 내보내기 폴더 관리)를 담당하며, File System Access API 우선 + `<a download>` 폴백 구조를 갖춘다. 전반적으로 구조가 견고하고 전체 백업(`_doExportWithGroups`)은 `flushCurrentState`로 다이어그램의 sections/notes/notesV2/collapsed까지 포함해 데이터 무결성 측면에서 양호하다. `index.html` onclick 핸들러와 전역 함수 연결, 단축키(Ctrl+S / Ctrl+Shift+S) 표시도 일치한다. 다만 SVG 내보내기가 접힌(collapsed) 엔티티 상태를 무시해 박스는 헤더 높이로 그려지면서 속성 행은 전부 그려지는 시각적 깨짐 버그가 발견되었으며, 클립보드/팝업/Markdown 이스케이프 등 견고성 개선 여지도 존재한다.

## 2. 발견된 오류 및 수정 결과

### [medium] SVG 내보내기가 접힌(collapsed) 엔티티의 속성을 그대로 렌더링 → 박스 밖으로 넘침

- **상태**: 수정완료
- **증상**: 엔티티를 접은(collapsed) 상태에서 SVG로 내보내면, 엔티티 박스는 `entityHeight(e)`가 `collapsedEntities`를 반영해 `HEADER_H`(=36)로만 그려지지만, 바로 아래 `e.attrs.forEach` 루프는 접힘 여부와 무관하게 모든 속성 행(PK/FK 배경, 배지, 컬럼명, 타입, 구분선)을 `ey+HEADER_H` 아래에 계속 그린다. 결과적으로 접힌 엔티티의 SVG 출력은 작은 헤더 박스 아래로 속성 텍스트들이 박스 경계를 벗어나 겹쳐 그려진다. 동일 데이터로 PNG 이미지(`downloadImage`)는 `drawEntity`를 재사용해 접힘 시 헤더만 그리고 return하므로(canvas.js `drawEntity`의 `isCollapsed` early return) PNG와 SVG 결과가 불일치한다. 즉 화면/PNG와 SVG가 어긋나는 명백한 출력 오류다.
- **원인**: `downloadSVG()`는 박스 높이 계산에만 `entityHeight(e)`(`collapsedEntities` 반영)를 쓰고, 속성 렌더링 루프 `e.attrs.forEach`에는 `collapsedEntities.has(e.id)` 검사를 두지 않았다. 캔버스 렌더 경로(`drawEntity`)는 접힘 시 속성 그리기를 건너뛰지만 SVG 경로에는 같은 분기가 누락되었다.
- **수정 내용**: SVG 내보내기에서 접힌(collapsed) 엔티티의 속성 렌더링을 건너뛰도록 `e.attrs.forEach` 앞에 `if (!collapsedEntities.has(e.id))` 조건을 추가했습니다.

## 4. 추천 사항

- **DDL 생성에도 다이어그램/엔티티 선택 및 옵션 노출**: `buildDDL`은 이미 `opts`(includeFK/includeIndex/includeComment)와 `entities` 인자를 받도록 설계되어 있으나 `generateDDL`은 항상 전역 `ENTITIES` 전체 + 모든 옵션 true로만 호출한다. DDL 모달에 FK/INDEX/COMMENT 토글 체크박스와 엔티티 선택을 추가하면 함수의 기존 유연성을 UI에서 활용할 수 있다.

- **내보내기 진행/결과에 대한 일관된 비동기 오류 처리**: `_writeExportFile`은 실패 시 false 반환 → 폴백 다운로드로 자연스럽게 이어지지만, JSON/백업 외의 SVG·Markdown·HTML 경로는 File System Access API를 거치지 않고 항상 `<a download>`만 쓴다. 모든 내보내기를 `_writeExportFile`(폴더 자동저장)+폴백 경로로 통일하면 사용자가 설정한 내보내기 폴더가 일관되게 적용된다.

- **PNG 내보내기를 toBlob+_writeExportFile 통합 및 고해상도(scale) 옵션 제공**: `downloadImage`는 `scale=1` 고정 + `toDataURL` 즉시 다운로드다. `devicePixelRatio` 기반 2x 옵션과 `toBlob` 비동기 다운로드, 그리고 폴더 자동저장(`_writeExportFile`) 연동을 추가하면 인쇄/문서용 고품질 출력과 일관된 저장 경험을 제공할 수 있다.

- **내보내기 단축키 동기화 점검 자동화 유지**: `index.html` 메뉴에는 JSON=Ctrl+S, 전체백업=Ctrl+Shift+S가 표시된다. `main.js`/`shortcuts.js`의 `matchSC` 매핑 및 단축키 모달(`shortcutsOverlay`) 표시가 export 함수와 어긋나지 않도록, 신규 내보내기 기능 추가 시 `CLAUDE.md`의 `feature-dev` 하네스로 단축키 3중 동기화를 계속 검증할 것을 권장한다.
