# 편집 ▸ 실행취소·초기화·모드 — 분석 보고서

## 1. 개요

이 메뉴(실행취소·초기화·보기/표기/스냅·섹션 모드 토글·전역 단축키)는 전반적으로 구조가 잘 잡혀 있다. undo/redo는 STORAGE_KEY 스냅샷 직렬화({diagrams, activeDiagramId, viewMode, notationStyle, gridSnap})를 기반으로 동작하고, restoreFromSnapshot은 뷰포트를 의도적으로 제외해 일관성을 지킨다. resetToDefault·복사/붙여넣기/복제/전체선택/방향키 이동/삭제 단축키도 대체로 saveState와 render를 정상 호출한다. 다만 영속성 무결성 측면에서 명백한 결함이 있다: gridSnap·notationStyle·viewMode는 모두 saveState 직렬화 대상이자 전체백업 대상인데, 이를 바꾸는 토글 함수와 보기 전환 진입점이 saveState()를 호출하지 않아 새로고침 시 변경이 유실되고 undo/redo 단계로도 기록되지 않는다.

## 2. 발견된 오류 및 수정 결과

### [medium] toggleGridSnap이 saveState()를 호출하지 않아 그리드 스냅 상태가 영속되지 않음

- **상태**: 수정완료
- **증상**: 메뉴바 '그리드 스냅' 또는 퀵바 ⊞ 버튼으로 gridSnap을 켜고/끈 뒤 다른 상태 변경(엔티티 이동 등) 없이 페이지를 새로고침하면 토글이 초기값(false)으로 되돌아간다. gridSnap은 saveState가 localStorage(STORAGE_KEY)에 기록하는 스냅샷의 일부이고 전체백업(exportFullBackup의 main)에도 포함되지만, toggleGridSnap은 변수만 바꾸고 saveState()를 호출하지 않으므로 localStorage에 반영되지 않는다. 동일한 이유로 undo/redo의 한 단계로도 잡히지 않아 '스냅 켜기'를 undo로 되돌릴 수 없다.
- **원인**: gridSnap이 영속 직렬화 대상(saveState/undo·redo)인데 이를 변경하는 toggleGridSnap에서 saveState() 호출이 누락되어 변경이 localStorage 및 undo 스택에 반영되지 않는다.
- **수정 내용**: toggleGridSnap 함수 끝에 saveState() 호출 추가하여 그리드 스냅 상태 영속화
- **파일**: js/ui.js

### [medium] toggleNotation이 saveState()를 호출하지 않아 표기법 상태가 영속되지 않음

- **상태**: 수정완료
- **증상**: 메뉴바 '크로우풋 표기' 또는 커맨드 팔레트로 notationStyle을 전환한 뒤 다른 상태 변경 없이 새로고침하면 표기법이 'simple'로 되돌아간다. notationStyle은 saveState 스냅샷과 전체백업(main)에 포함되는 영속 상태인데, toggleNotation은 변수만 토글하고 saveState()를 호출하지 않아 localStorage에 반영되지 않고 undo/redo 단계로도 기록되지 않는다.
- **원인**: notationStyle이 영속 직렬화 대상인데 이를 변경하는 toggleNotation에서 saveState() 호출이 누락되었다.
- **수정 내용**: toggleNotation 함수 끝에 saveState() 호출 추가하여 표기법 상태 영속화
- **파일**: js/ui.js

### [medium] 논리/물리 보기 전환(setViewMode) 메뉴가 saveState()를 호출하지 않아 viewMode가 영속되지 않음

- **상태**: 수정완료
- **증상**: 메뉴바의 '논리 보기'/'물리 보기'(및 퀵바 논리/물리 버튼, index.html:169-170)는 setViewMode('logical'|'physical')만 호출한다. viewMode는 saveState 스냅샷·전체백업에 포함되는 영속 상태인데, 이 onclick들이 saveState()를 부르지 않으므로 다른 변경 없이 새로고침하면 마지막으로 저장된 viewMode로 되돌아간다. 참고: setViewMode 자체는 init(main.js:18)·loadState·restoreFromSnapshot(state.js:142)·importFullBackup(import.js:53)에서도 호출되므로 함수 내부에 saveState()를 넣으면 로드/복원 중 undo 스택이 오염된다. 따라서 사용자 토글 진입점(메뉴바, 퀵바 169-170행)에서만 saveState()를 호출해야 한다.
- **원인**: viewMode가 영속 직렬화 대상이지만 보기 전환 진입점에서 saveState() 호출이 누락되었고, setViewMode가 로드/복원 경로와 공유되어 내부에 saveState를 넣을 수 없어 메뉴 진입점 보강이 필요하다.
- **수정 내용**: 논리/물리 보기 전환 메뉴 onclick에 saveState() 호출을 추가하여 viewMode가 LocalStorage에 영속되도록 수정
- **파일**: index.html

### [low] 삭제(del) 단축키 핸들러에 도달 불가능한 중복 selectedSections 분기

- **상태**: 수정실패
- **증상**: keydown del 처리에서 selectedSections.size > 0 분기가 두 번 연속 등장한다. 첫 번째 분기(인라인 SECTIONS.splice 후 selectedSections.clear())가 항상 먼저 실행되어 return 하므로, 그 뒤의 두 번째 selectedSections.size > 0 분기(deleteSection을 호출하는 코드)는 절대 실행되지 않는 죽은 코드다. 현재 기능은 첫 분기로 정상 동작하지만, 원래 deleteSection()을 호출하려던 의도가 무시되고 있으며 유지보수 시 혼동을 유발한다.
- **원인**: 동일 조건의 분기가 중복 작성되어 첫 분기에서 return 되므로 두 번째 deleteSection 분기가 데드 코드가 되었다.
- **수정 내용**: ANCHOR에 명시된 두 번째 selectedSections.size > 0 중복 분기(deleteSection 호출)가 파일에 존재하지 않음. 현재 파일은 이미 단일 분기만 있어 REPLACEMENT 상태와 동일하므로 적용 불필요.
- **파일**: js/main.js

## 4. 추천 사항

- **영속 상태 토글의 saveState 호출을 회귀 테스트로 보호**: viewMode/notationStyle/gridSnap처럼 saveState 직렬화 대상이면서 토글 UI를 가진 상태는 '토글→localStorage 반영'을 검증하는 경량 테스트(또는 체크리스트)를 두면 같은 누락의 재발을 막을 수 있다. CLAUDE.md의 단축키·백업 동기화 검증 하네스와 같은 맥락으로 '영속 토글 검증' 항목을 추가 권장.
- **단축키 정의의 단일 출처화**: main.js의 keydown 처리, shortcuts.js의 matchSC, index.html shortcutsOverlay 표시가 세 곳에 흩어져 있어 ctrl+shift+z 같은 하드코딩이 끼어든다. 단축키 정의를 shortcuts.js 한 곳으로 모으고 모달 표시도 그 정의에서 렌더링하면 표시-동작 불일치를 구조적으로 예방할 수 있다.
- **undo/redo 스택을 비파괴적 baseline 기준으로 일관 운영**: 현재 baseline 확보(main.js:22), undoStack.length<=1 가드(undo) 등이 흩어져 있다. 스택 진입/이탈 로직을 saveState/undo/redo 한 모듈로 캡슐화하고, '뷰포트는 undo 제외'라는 규칙을 주석이 아닌 직렬화 헬퍼로 강제하면 회귀에 강해진다.
