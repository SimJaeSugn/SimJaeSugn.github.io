# 보기 ▸ 배치·정렬 — 분석 보고서

## 1. 개요

`js/layout.js`는 "보기 ▸ 배치·정렬" 메뉴의 핵심 로직을 담당하며, 섹션 모드·자동 배치 3종·관계선 최적화 V1/V2·정렬 8종을 구현한다. `index.html`의 `onclick` 핸들러(`autoLayout`, `autoOptimizeRelations`, `autoOptimizeRelationsV2`, `alignEntities`, `toggleSectionMode`)와 정확히 연결되어 있고, 호출하는 외부 함수(`entityHeight`, `faceAnchor`, `buildFullWpts`, `obstacleOnSeg`, `centerOnEntities`, `_runFaceSpacingPass`, `_fixEntityCrossingsForRel`, `render`, `saveState` 등)도 모두 존재한다. 함수 연결·상태 저장(`saveState` 호출)·빈 배열/단일 선택 가드는 전반적으로 잘 갖춰져 있다. 다만 비동기 최적화 루프(rAF)의 재진입 방지 부재, 미사용 죽은 코드(`_recomputeRelWpts`), 다이어그램 전환 중 비동기 `saveState` 경쟁 등 견고성 이슈가 일부 발견되었다.

## 2. 발견된 오류 및 수정 결과

### [low] 섹션 이름 입력에서 Escape를 눌러도 취소되지 않고 이름이 커밋됨

- **상태**: 오류아님
- **증상**: `showSectionNameInput`에서 섹션 이름을 편집할 때 Escape를 누르면 입력을 취소(되돌리기)해야 하지만, Escape 핸들러가 `wrap.remove()`로 포커스된 input을 DOM에서 제거하는 순간 브라우저가 blur 이벤트를 발생시키고, blur에 바인딩된 `commit()`이 실행되어 사용자가 입력 중이던 값으로 `section.name`이 그대로 저장된다는 주장이다. 또한 Enter 시에도 `commit()` 내부의 `wrap.remove()`가 다시 blur→`commit()`을 유발해 `render()`/`saveState()`가 중복 실행된다는 주장도 포함된다.
- **원인**: `commit`이 단 한 번만, 그리고 '확정 경로'에서만 실행되도록 보장하는 가드가 없다는 주장. Escape 경로는 `commit`을 부르지 않고 `wrap`만 제거하지만, 포커스된 요소를 detach하면 blur가 발생하여 blur 리스너의 `commit`이 호출된다는 전제를 기반으로 하며, `commit`/`cancel`을 1회로 제한하는 플래그가 필요하다는 제안이다.
- **수정 내용**: 위 주장의 핵심 전제(포커스된 input을 `wrap.remove()`로 DOM에서 분리하면 브라우저가 동기적으로 blur 이벤트를 발생시켜 blur에 바인딩된 `commit()`이 호출된다)가 실제 최신 브라우저 동작과 어긋난다. 현행 브라우저(Chrome/Firefox/Safari)와 WHATWG 포커스 fixup 규칙상, 포커스된 활성 요소를 DOM에서 제거하면 `activeElement`는 `body`로 되돌아가지만 제거된 노드에는 blur/focusout 이벤트가 발생하지 않는다. 따라서 Escape 경로는 정상적으로 취소되며(이름이 커밋되지 않음), Enter 경로에서도 `commit`/`render`/`saveState`의 중복 실행이 발생하지 않는다. 동일한 idiom이 `js/ui.js`의 `showRelLabelInlineEdit`에도 반복 사용되며, 개발자가 이 패턴을 의도적으로 활용하고 있음을 뒷받침한다. 제안된 `done` 플래그 수정은 무해하지만 존재하지 않는 문제를 해결하는 것이므로 수정이 불필요하다.
- **파일**: `js/layout.js`

## 4. 추천 사항

- **최적화/배치 작업에 취소(ESC) 및 중단 버튼 제공**: `layoutProgress` 모달은 진행만 표시하고 사용자가 중단할 수 없다. 대형 다이어그램에서 V2가 오래 걸릴 때 ESC 또는 '취소' 버튼으로 rAF 루프를 안전하게 멈추고 직전 상태로 되돌리는 기능을 추가하면 사용성이 크게 향상된다.

- **배치/최적화 결과를 단일 undo 단위로 묶기**: 현재는 비동기 루프 종료 시점에 `saveState()` 1회만 수행되어 대체로 단일 undo로 동작하지만, 재진입/중복 실행 시 여러 `saveState`가 끼어들 수 있다. 작업 시작 시 스냅샷을 한 번 캡처하고 종료 시 1회만 커밋하는 '트랜잭션' 패턴을 도입해 undo 단위 일관성을 보장하길 권장한다.

- **자동 배치 후 화면 맞춤(fit-to-screen) 옵션 추가**: `centerOnEntities`는 좌상단 기준 패닝만 수행하고 scale을 조정하지 않아 큰 배치 결과가 화면을 벗어날 수 있다. 배치 직후 전체가 보이도록 scale까지 자동 조정하는 fit 동작(또는 사용자 토글)을 제공하면 계층/원형 배치 결과 확인이 쉬워진다.

- **정렬·배치 동작에 단축키 부여 및 단축키 모달 동기화**: 현재 `SC_DEFAULTS`(`js/shortcuts.js`)에는 정렬/배치/최적화 단축키가 없다. 자주 쓰는 정렬(좌/우/상/하/중앙)과 자동 배치에 단축키를 추가한다면, `SC_DEFAULTS`·`main.js` keydown·`index.html` 단축키 모달 세 곳을 함께 갱신하여 동기화 규칙을 지켜야 한다.
