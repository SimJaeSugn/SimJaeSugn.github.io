# 도구 ▸ 정규화 진단 — 분석 보고서

## 1. 개요

정규화 진단 메뉴(js/normalize.js)는 (1) 반복 컬럼 패턴(이름 끝 숫자 2개 이상 → 1NF 위반 후보), (2) N:M 직접 연결을 검사해 `_normWarnings`(entityId→경고 배열)와 `_normActive` 전역에 담고, `render()` 시 canvas.js:952 에서 엔티티 헤더에 ⚠ 배지를 그리며, 결과 모달(`openNormResultModal`)과 배지 제거(`clearNormDiagnosis`)를 제공한다. 검사 로직 자체와 escHtml 이스케이프, 모달 캐싱은 대체로 견고하다. 다만 `_normWarnings`/`_normActive`가 다이어그램 전환·undo/redo 시점에 초기화되지 않아 다른 다이어그램에 잘못된 ⚠ 배지가 남는 상태 누수 버그가 있고, README에 명시된 "배지 hover 시 경고 툴팁 표시" 기능이 실제로는 구현되어 있지 않다. 전반적으로 진단 알고리즘은 동작하나 전역 상태 수명 관리와 문서-구현 일치 측면에서 결함이 있다.

## 2. 발견된 오류 및 수정 결과

### [medium] 정규화 진단 배지 상태가 다이어그램 전환/undo·redo 시 초기화되지 않아 엉뚱한 다이어그램에 ⚠ 배지가 남음

- **상태**: 수정완료
- **증상**: 다이어그램 A에서 '정규화 진단'을 실행하면 `_normActive=true`, `_normWarnings(entityId→경고)`가 채워지고 헤더에 ⚠ 배지가 그려진다. 이후 switchDiagram/다이어그램 삭제/undo/redo 등으로 `loadDiagramIntoWorkspace`가 호출되어도 이 두 전역이 그대로 유지된다. canvas.js:952 의 배지 렌더는 `_normActive && _normWarnings[e.id]`만 보므로, 다른 다이어그램에 같은 entityId(예: 직접 입력한 'entity_user' 같은 id, 또는 다이어그램간 엔티티 복사로 동일 base id 보유, 동일 ms 생성으로 인한 id 충돌)가 존재하면 진단한 적 없는 엔티티에 잘못된 ⚠ 배지가 표시된다. id가 안 겹치면 배지는 안 보이지만 `_normActive`가 true로 남아 다음 동작과 혼란을 유발하고, '배지 지우기'를 누르지 않는 한 잔존한다. 재현: 다이어그램 A 진단(이상 있음) → 다이어그램 B로 전환 → B의 동일 id 엔티티에 ⚠ 배지가 잘못 표시됨.
- **원인**: `_normWarnings`/`_normActive`는 normalize.js 의 전역(파일 스코프 let)으로, 다이어그램 작업 배열(ENTITIES 등)과 함께 교체되지 않는 휘발성 진단 상태다. 모든 다이어그램 로드의 단일 진입점인 `loadDiagramIntoWorkspace`에서 진단 상태를 초기화하지 않기 때문에, entityId 기반 `_normWarnings`가 새 다이어그램의 엔티티와 잘못 매칭된다.
- **수정 내용**: `loadDiagramIntoWorkspace()` 내에서 다이어그램 전환·undo·redo 시 `_normActive`와 `_normWarnings`를 초기화해 정규화 진단 배지가 다른 다이어그램에 잔류하는 버그를 수정
- **파일**: js/state.js

### [low] README 에 명시된 '경고 배지 hover 시 툴팁 표시'가 구현되어 있지 않음

- **상태**: 수정완료
- **증상**: README 11절(라인 398-399)은 '진단 후 문제가 있는 엔티티 헤더 우측에 ⚠ 주황색 배지가 표시되고, 배지에 마우스를 올리면 구체적인 경고 내용이 툴팁으로 표시됩니다.'라고 사양을 명시한다. 그러나 `updateTooltip()`은 헤더 hover 시 `ent.description`만 표시하며 `_normWarnings`를 전혀 참조하지 않는다. 따라서 진단이 활성화되어 ⚠ 배지가 떠 있어도 배지(헤더)에 마우스를 올리면 description이 없는 엔티티는 툴팁이 아예 안 뜨고, 경고 내용은 결과 모달을 다시 열기 전까지 확인할 수 없다.
- **원인**: 정규화 진단 기능이 배지 렌더(canvas.js)와 결과 모달(normalize.js)만 구현하고, 헤더 hover 툴팁 경로(ui.js `updateTooltip`)에 `_normWarnings` 분기를 추가하지 않았다. 헤더 영역 hover 처리는 `ent.description` 유무로만 분기하므로 진단 경고가 표시되지 않는다.
- **수정 내용**: `updateTooltip` 헤더 영역 hover 시 `_normWarnings` 경고가 있으면 우선 표시하는 로직 추가
- **파일**: js/ui.js

## 4. 추천 사항

- **진단 결과를 다이어그램별로 캐싱하고 재진단 단축키 제공**: `_normWarnings`를 다이어그램 객체(또는 id 매핑)에 보관해 전환 시 해당 다이어그램의 마지막 진단 결과를 복원하면 상태 누수도 막고 사용자 경험도 개선된다. 더불어 진단/배지 제거에 단축키를 부여하고 shortcuts.js + index.html shortcutsOverlay에 동기화 표기하면 다른 도구 기능과 일관된다(현재 정규화 진단은 단축키 미할당으로 동기화 이슈는 없음).
- **추가 정규화 규칙 확장**: PK 부재 엔티티, FK가 PK 일부만 참조하는 부분 함수 종속(2NF), 비키 속성 간 이행 종속(3NF) 후보, 동일 의미 컬럼 중복 등 규칙을 추가하면 진단 가치가 커진다. 규칙별 심각도(경고/정보)와 켜고 끄기 옵션을 모달에 제공하면 활용도가 높아진다.
- **진단 결과를 클릭하면 해당 엔티티로 포커스/이동**: 결과 모달의 각 엔티티 항목 클릭 시 `setFocusEntity` 또는 캔버스 패닝으로 해당 엔티티를 화면 중앙에 비추면, 다수 경고를 빠르게 점검할 수 있어 UX가 개선된다.
