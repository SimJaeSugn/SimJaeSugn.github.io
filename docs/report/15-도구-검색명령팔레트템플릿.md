# 도구 ▸ 검색·명령 팔레트·템플릿 — 분석 보고서

## 1. 개요

이번 분석은 "도구 ▸ 검색·명령 팔레트·템플릿" 메뉴 영역에서 발견된 오류만을 대상으로 수행한 bisect 검사 결과다. 총 4건의 오류가 식별되었으며, 그 중 3건은 수정이 완료되었고 1건은 분석 결과 실제 오류가 아님이 확인되었다. 개선 제안 및 추천 사항은 해당 사항이 없다.

## 2. 발견된 오류 및 수정 결과

### [낮음] 커맨드 팔레트 복제 명령이 빈 함수라 동작하지 않음

- **상태**: 수정완료
- **증상**: Ctrl+K 팔레트에서 복제를 선택하면 action이 빈 화살표 함수라 아무 일도 안 일어난다. 동일 기능 단축키 Ctrl+D는 선택 엔티티와 섹션을 복제하고 saveState까지 하는데 팔레트 경로만 조용히 실패한다.
- **원인**: 복제 로직이 main.js 키보드 핸들러에 인라인으로만 존재하고 전역 함수로 추출되지 않아 CMD_LIST action이 빈 스텁으로 방치됨.
- **수정 내용**: 커맨드 팔레트 복제 action의 빈 함수를 엔티티·섹션 복제 실제 로직으로 교체
- **파일**: js/ui.js

### [낮음] 검색 점프 jumpToEntity가 좌측 도킹 시 좌표 보정 누락

- **상태**: 오류아님
- **증상**: 검색 결과 클릭 점프 시 vx 계산에 _qbLeftOff 보정이 빠져 좌측 도킹 시 엔티티가 퀵바 너비만큼 우측으로 치우친다.
- **원인**: 좌측 도킹 보정 도입 시 jumpToEntity의 vx 계산에는 반영되지 않음.
- **수정 내용**: 거짓 양성으로 판정되어 수정하지 않음. renderNow()(js/canvas.js:1749~1786)를 확인한 결과, 좌측 도킹 보정은 vx가 아니라 캔버스 DOM 요소의 CSS marginLeft 및 축소된 canvas.width로 처리된다. 따라서 엔티티 중앙 정렬용 vx는 이미 축소된 canvas.width 기준으로 계산하는 현재 코드(`vx = canvas.width/2 - (en.x+W/2)*scale`)가 올바르다. minimapNavigateTo(ui.js:597)도 동일 방식을 사용하여 일관된 관례를 따르고 있으며, 제안된 수정을 적용하면 오히려 엔티티가 퀵바 너비만큼 좌측으로 치우쳐 잘못 배치된다.
- **파일**: E:\04.개발환경\python\98.ETC\SimJaeSugn.github.io\js\ui.js

### [낮음] 팔레트 라벨 오타로 템플릿 검색 시 명령 미노출

- **상태**: 수정완료
- **증상**: CMD_LIST의 컬럼 탬플릿 관리 라벨 오타로 템플릿 검색 시 결과에 안 나온다.
- **원인**: 라벨 부분문자열 매칭에 의존하므로 표기 불일치가 검색 누락으로 이어짐.
- **수정 내용**: 팔레트 '컬럼 탬플릿 관리' 오타를 '컬럼 템플릿 관리'로 수정
- **파일**: js/ui.js

### [낮음] 팔레트 전체 선택이 섹션 미선택으로 단축키와 불일치

- **상태**: 수정완료
- **증상**: 전체 선택 action이 ENTITIES만 선택하고 SECTIONS와 사전 clear를 누락해 단축키와 결과가 다르다.
- **원인**: 팔레트 action이 main.js selAll 로직과 별도로 축약 작성됨.
- **수정 내용**: 팔레트 전체 선택 action에 selectedSections.clear() 및 SECTIONS 순회 추가로 섹션도 함께 선택
- **파일**: js/ui.js
