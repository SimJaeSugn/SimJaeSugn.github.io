# 캔버스 ▸ 섹션·메모·미니맵 — 분석 보고서

## 1. 개요

이 메뉴(섹션·메모·미니맵)는 전반적으로 잘 구조화되어 있다. 섹션은 hitTestSectionResize/Label로 8방향 리사이즈·라벨 드래그·이름변경·삭제를 지원하고, 메모는 V1(스티커, Text/MD 탭 + 캔버스 렌더 + 마크다운 DOM 오버레이)과 V2(핀고정/색상/태그 contentEditable 카드)로 분리되어 있다. 미니맵은 OffscreenCanvas Worker(폴백 포함)로 렌더링되고 클릭/드래그로 뷰포트 이동을 지원한다. 상태 영속성은 flushCurrentState→saveState 경로로 SECTIONS/NOTES/NOTES_V2가 다이어그램 객체에 직렬화되어 undo/redo·localStorage·전체백업(exportFullBackup)에 모두 포함된다. 다만 기본 COLOR 팔레트 누락으로 인한 테마 미적용 시 렌더 오류와 엔티티 접기 상태 유실 버그 두 가지가 실제 문제로 확인되었다.

## 2. 발견된 오류 및 수정 결과

### [medium] 기본 COLOR 팔레트에 sectionLabelBg·ac_y 누락 → 테마 미적용 시 섹션 라벨/강조색 렌더 오류

- **상태**: 수정완료
- **증상**: 저장된 테마가 없는 첫 방문 시 loadSavedTheme()가 applyTheme를 호출하지 않아 기본 COLOR 객체(config.js 126~150)가 그대로 사용된다. 그런데 기본 COLOR에는 sectionLabelBg와 ac_y 키가 없다(테마 정의에만 존재). 결과적으로 drawSectionShape의 'ctx.fillStyle = COLOR.sectionLabelBg'가 undefined로 세팅되어(캔버스에서 무시됨) 섹션 라벨 바가 직전 fillStyle(섹션 본문색 pal.bg)로 채워져 잘못된 색으로 보인다. 또한 볼륨 라벨(_volumeBorder의 color: COLOR.ac_y), NOT NULL 'N' 배지, 정규화 경고 배지가 모두 undefined 색이 되어 강조색이 깨진다. 아무 테마나 한 번 적용하면 자동 복구되지만, 신규 사용자는 첫 화면에서 색 오류를 보게 된다.
- **원인**: 기본 COLOR 객체(config.js)에 캔버스 전용 키 sectionLabelBg, ac_y가 정의되지 않았고, 초기화 시 기본 테마를 강제 적용하지 않아 COLOR가 불완전한 채로 첫 render가 실행됨.
- **수정 내용**: 기본 COLOR 객체의 selFill 다음에 ac_y·sectionLabelBg 두 속성을 추가하여 테마 미적용 시 섹션 라벨/강조색 렌더 오류 수정
- **파일**: js/config.js

### [medium] 엔티티 헤더 접기 토글이 saveState 누락(flushCurrentState만 호출) → 새로고침·undo 시 접기 상태 유실

- **상태**: 수정완료 *(정정 — 형제 메뉴 에이전트가 이미 적용)*
- **증상**: 캔버스 헤더 더블클릭으로 엔티티 접기/펴기를 토글할 때 flushCurrentState()와 render()만 호출하고 saveState()를 호출하지 않는다. flushCurrentState는 현재 작업 배열을 활성 다이어그램 객체(d.collapsed)에만 복사할 뿐 localStorage 기록이나 undo 스택 push를 하지 않는다. 따라서 접기 상태는 (a) 페이지 새로고침 시 localStorage의 이전 스냅샷으로 복원되며 사라지고, (b) undo/redo로 되돌릴 수 없다. 접기 상태는 미니맵 엔티티 높이 계산(collapsed)에도 반영되므로 미니맵 표시에도 영향을 준다.
- **원인**: 상태 변경 후 영속화 경로인 saveState() 대신 메모리 플러시만 하는 flushCurrentState()를 호출함. 다른 캔버스 조작들은 모두 'render(); saveState();' 패턴을 쓰는 것과 불일치.
- **수정 내용**: **수정 완료.** js/canvas.js dblclick(엔티티 헤더 접기/펴기) 핸들러에서 `flushCurrentState()` 호출을 제거하고 `render(); saveState();` 패턴으로 교체해 접기 상태를 localStorage·undo/redo에 영속화했다(`saveState()` 가 내부에서 `flushCurrentState()` 를 먼저 호출하므로 기존 플러시는 유지). *(집계 정정: 이 수정은 자동 워크플로의 캔버스 렌더링 메뉴 에이전트가 동일 버그를 안전 수정으로 적용한 것으로, 메뉴별 상태 집계가 타 메뉴의 고유 ID(gid)로 매칭되어 본 보고서에 stale 하게 '수동검토필요'로 남았던 것을 후속 정정함.)* 〔이하 최초 분석 기록〕 js/canvas.js의 dblclick 핸들러(2275-2281행)에서 헤더 더블클릭으로 엔티티 접기/펴기를 토글한 뒤 flushCurrentState()와 render()만 호출하고 saveState()를 호출하지 않는다. 영속성 흐름 추적 결과, flushCurrentState()(state.js:58-68)는 collapsedEntities를 활성 다이어그램 객체의 d.collapsed에 메모리 복사만 한다. 유일한 영속화 경로는 saveState()(state.js:82-96)뿐이며, saveState는 내부에서 flushCurrentState()를 먼저 부른 뒤 diagrams를 직렬화해 undoStack에 push하고 localStorage.setItem으로 저장한다. 자동저장/setInterval/beforeunload 영속화 경로는 존재하지 않는다. 새로고침 시 loadState→loadDiagramIntoWorkspace(state.js:76)가 localStorage 스냅샷의 d.collapsed를 다시 읽어 collapsedEntities를 복원하므로, 토글이 localStorage에 기록되지 않은 경우 이전(접기 전) 스냅샷으로 복원되어 접기 상태가 유실된다. canvas.js의 다른 캔버스 조작들(1893, 1968, 2412, 2423행)과 동일한 'render(); saveState();' 패턴으로 교체하면, saveState()가 내부에서 flushCurrentState()를 먼저 호출하므로 기존 플러시 동작은 유지되고 undo 스택 push와 localStorage 영속화가 추가된다.
- **파일**: js/canvas.js

## 4. 추천 사항

- **미니맵 클릭 이동 시 뷰포트 사각형과 퀵바 좌측 도킹 오프셋 일관성 검증 테스트 추가**: 미니맵 뷰포트 사각형은 -vx/scale, canvas.width/scale 기준으로 그려지고 캔버스 marginLeft(_qbLeftOff)와 정합하도록 설계되어 있다. 퀵바를 좌측 도킹한 상태에서 미니맵 클릭→중앙 이동이 실제 화면 중앙과 맞는지 자동/수동 회귀 테스트를 추가하면 좌표 변환 리그레션을 조기에 잡을 수 있다.
- **엔티티 접기 상태를 undo/redo 대상으로 명확히 정의**: collapsedEntities는 d.collapsed로 직렬화되어 스냅샷에 포함되므로, 접기 토글 시 saveState를 호출하면 자연스럽게 undo/redo·영속화가 작동한다. 접기를 undo 대상에 포함할지 정책을 문서(README 사양)에 명시하고, 트리 패널 등 다른 접기 경로도 동일하게 saveState를 타도록 통일 권장.
- **메모/섹션 추가에 대한 단축키 및 툴바 진입점 보강**: 현재 메모 추가는 우클릭 컨텍스트 메뉴로만 가능하고 섹션은 Ctrl+드래그/섹션 모드로만 진입한다. 자주 쓰는 기능이라면 퀵바 버튼이나 단축키(shortcuts.js + 단축키 모달 동기화)를 추가해 발견성을 높이는 것을 권장한다.
- **기본 테마 강제 적용으로 COLOR 완전성 보장**: 오류 1의 근본 해결을 위해 초기화 시 저장 테마가 없으면 기본 테마(dark)를 명시적으로 applyTheme('dark', false)하도록 하여 COLOR 객체가 항상 모든 키(sectionLabelBg, ac_y 등)를 갖도록 보장하면, 향후 테마에 키가 추가될 때 기본 COLOR 누락 버그가 재발하지 않는다.
